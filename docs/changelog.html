Changelog

What's added in each version of Freally Central. Newest first.

In development Phase 2 — live release data & real download counts

0.2.0 — Phase 2

Every card is now wired to GitHub's real numbers. Central resolves each app's latest release from the GitHub Releases API and shows the live version and date, per-OS and total download counts, and a brand-wide grand total — always the real figures, never seeded, and hidden (not faked) when the API is offline or rate-limited.

  • Live release data — latest version + date per app, resolved from the Releases API against each app's per-OS asset patterns.
  • Real download counts — per-OS and total per app, plus a brand-wide grand total, straight from GitHub's download_count.
  • Changelog / What's New viewer — the latest release's notes shown verbatim in-app, with fully localized chrome and links out to the full changelog.
  • 18-language UI — all new strings localized across the 18 locales; the CI parity lint stays green.
In development Phase 1 — the shell & the catalog grid

0.1.0 — Phase 1

The Tauri v2 + React + Rust app itself: a dark/light themed window and a product-card grid driven by the hosted catalog manifest, with an offline fallback so the grid always renders.

  • Product-card grid + detail view — big icon, name, tagline, description, features, and start date, with a Type filter and a search box.
  • First-run EULA gate — shown once on first use; acceptance is remembered.
  • Settings — About, What's New, and a self-hosted signed check-for-updates flow.
  • 18-language UI — the Fluent i18n system and 18 locales with a CI parity + coverage lint.
In development Phase 0 — repo, CI & the 3-OS release pipeline

Unreleased — Phase 0

Standing up the project before any product code: the GitHub repository, a Windows/macOS/Linux CI matrix, and the signed release pipeline that builds installers for all three OSes and publishes the updater endpoint. The product roadmap and build-prompts guide define the phases that follow (the grid, real download counts, install/update status, downloads with a live progress bar, hands-off installs, and the embeddable panel).

  • Project scaffold — README, license (proprietary, All Rights Reserved), EULA, feature roadmap, build-prompts guide, and the product catalog manifest.
  • This docs site — the landing/download page (with a real per-OS GitHub download counter), this changelog, and the documentation hub, matching the other Freally sites.
  • Coming with Phase 0 — signed installers for Windows/macOS/Linux and the self-hosted updater.
