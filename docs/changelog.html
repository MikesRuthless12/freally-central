Changelog

What's added in each version of Freally Central. Newest first.

v0.7.0 Phase 7 — polish, distribution & auto-update

0.7.0 — Phase 7

Central now ships the way the rest of the brand does: a signed self-updater that resolves on every OS, an accessibility & reduced-motion pass, and a public download page with real per-OS counts.

  • In-app auto-update, all three OSes — the release now publishes one consolidated signed latest.json covering Windows, macOS (Intel + Apple Silicon) and Linux, so Settings → Check for updates resolves and verifies every download against the key baked into the app.
  • Accessibility — dialogs move, trap and restore keyboard focus and close on Escape, the first-run agreement is fully keyboard-operable, progress and status changes are announced to screen readers, and every control shows a visible keyboard focus ring.
  • Reduced motion — animations and transitions are neutralized for anyone whose system asks for reduced motion.
  • Light & dark — both themes verified end to end, down to native scrollbars and form controls that now follow the chosen theme.
  • Public download page — per-OS installer buttons for Windows, macOS and Linux beside the real, auto-incrementing GitHub download counter.
v0.6.0 Phase 6 — "Central inside", the embeddable panel

0.6.0 — Phase 6

The whole hub is now an embeddable panel any Freally app can ship under a "More Freally apps" entry — the card grid, detail view, live release data, install detection, and the verified download → silent-install flow, with no copy of the business logic. Freally Capture ships it first.

  • One implementation, everywhere — hosts vendor the Central repo as a git submodule; one pinned commit supplies both the React panel and the Rust engine crate, and Central itself runs on exactly the same code.
  • Freally Capture embed — Help → More Freally apps opens the panel inside Capture, themed by Capture's own tokens (light/dark follows the host).
  • Localized through the host — the panel's strings ship as fcp-* catalogs in all 18 languages and load into the host app's own Fluent bundles.
  • The trust gate rides along — required published SHA-256, the compile-time owner allowlist, and the install-time re-hash apply identically inside every host.
  • EMBEDDING.md — the drop-in checklist for rolling the panel out across the rest of the brand.
v0.5.0 Phase 5 — hands-off silent install

0.5.0 — Phase 5

Central now installs hands-off. One click on Download & install all (or a card's Install/Update) carries an app through download, verification, and a silent install with the installer's own defaults — zero wizard clicks where the OS allows it, at most one elevation consent per batch, and the same live bar running to 100.00% across both stages.

  • Silent installers per OS — Windows NSIS /S (per-user, no UAC; msiexec /qn as the MSI fallback), macOS dmg mount + copy into Applications, Linux AppImage placement or deb/rpm behind a single pkexec consent for the whole batch.
  • Only verified official installers ever execute — a matching published SHA-256 is required, the release owner must be in an allowlist compiled into the app, and the file is re-hashed at install time; a tampered or unverified file is refused and the refusal is shown honestly.
  • Install progress in the same bar — a determinate staged estimate that only claims 100% on the installer's real success, on the card, the detail view, and the batch aggregate.
  • Installed ✓ and Open — the badge flips on real success (confirmed by a detection re-probe) and the detail view launches the app from what its installer actually registered.
  • Honest cancel — pending installs cancel; a running installer is never killed mid-run and reports its real outcome.
  • 18-language UI — all new strings localized across the 18 locales; the CI parity lint stays green.
v0.4.0 Phase 4 — downloads, the live progress bar & Download All

0.4.0 — Phase 4

Central now downloads installers itself. A native engine streams the right installer for your OS with a precise two-decimal percent (e.g. 38.92%) computed from real bytes — landing on exactly 100.00% — and every file is verified (size + SHA-256) before it is kept. One click downloads the whole family.

  • Download engine — streams each app's per-OS installer from its GitHub release to a temp folder; interrupted downloads resume where they left off; only github.com/…/releases/download/… URLs are ever fetched.
  • Verification before use — the byte count must match the published size exactly and the SHA-256 digest must match when published; a short or corrupted download is refused and discarded.
  • Live progress bar — on the card and the detail view, bound 0 → 100% to real bytes, with Cancel and a resuming Retry.
  • Download All — queues every available app with bounded concurrency, an aggregate bar plus per-app bars, Cancel all, and per-app failure isolation with an honest summary.
  • Simpler top bar — the search box was removed (seven apps don't need one); the Type filter stays.
  • 18-language UI — all new strings localized across the 18 locales; the CI parity lint stays green.
v0.3.0 Phase 3 — install detection & status badges

0.3.0 — Phase 3

Central now knows what's on your machine. It detects the installed version of each app per OS and badges every card Installed ✓ / Update available / Not installed, computed by a semver-aware comparison to the latest release — honest by construction, a version is shown only when an installer actually wrote it.

  • Installed-version detection — Windows uninstall registry keys, macOS Info.plist, and Linux dpkg/rpm packages (with an AppImage-filename fallback), mapped to each catalog app. Runs off the UI thread; cached with a manual Refresh.
  • Status badges — Installed ✓ / Update available / Not installed on every card and the detail view.
  • Status-aware action — the detail view's primary button reflects status (Install / Update / Re-download), routing to the verified download source.
  • Desktop shortcut — the Windows installer now creates a Desktop icon on install.
  • 18-language UI — all new strings localized across the 18 locales; the CI parity lint stays green.
In development Phase 2 — live release data & real download counts

0.2.0 — Phase 2

Every card is now wired to GitHub's real numbers. Central resolves each app's latest release from the GitHub Releases API and shows the live version and date, per-OS and total download counts, and a brand-wide grand total — always the real figures, never seeded, and hidden (not faked) when the API is offline or rate-limited.

  • Live release data — latest version + date per app, resolved from the Releases API against each app's per-OS asset patterns.
  • Real download counts — per-OS and total per app, plus a brand-wide grand total, straight from GitHub's download_count.
  • Changelog / What's New viewer — the latest release's notes shown verbatim in-app, with fully localized chrome and links out to the full changelog.
  • 18-language UI — all new strings localized across the 18 locales; the CI parity lint stays green.
